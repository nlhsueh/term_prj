# 期末專案管理系統 (Course Project Management System) - 規格書

## 1. 專案背景與目標
本系統旨在協助大學程式語言課程的教授管理期末專案。主要目標是自動化分組收集、計劃書上傳以及期末評分管理。

## 2. 使用者角色與功能

### 學生 (Students)
- **分組登記**：由隊長選擇組員學號。
- **計劃書提交**：上傳 PDF 格式的專案計劃書。
- **進度查看**：確認分組與計劃書是否已成功繳交。

### 教師/助教 (Admin/Professor/TA)
- **學生清單管理**：匯入學生名單(學號與姓名)。
- **分組進度追蹤**：查看哪些學生尚未分組，哪些組別尚未繳交計劃書。
- **計劃書審閱**：下載或線上預覽學生的計劃書。
- **密碼管理**：手動重設特定學生的密碼（還原為學號後 4 碼）。
- **期末評分**：在評分介面直接輸入各組的報告與展示分數。
- **數據導出**：將最終的分組與成績導出為 CSV/Excel。

## 3. 核心流程

### 第一階段：期中 (Mid-term)
1. 老師登入系統設定課程資訊，包含名稱，分組設定時程，計畫初稿提交時程，期末報告提交時程。
2. 老師口頭要求學生登入，修改密碼。
3. 隊長選擇組員並設定專案名稱，選擇時呈現學生姓名及學號後兩碼。
4. 系統確認組員學號是否重疊或不存在。
5. 學生登入系統確認分組。
6. 學生上傳計劃書初稿。
7. 老師審核計劃書初稿。
8. 學生修改計劃書初稿。

### 第二階段：期末 (Final)
1. 學生上傳期末報告，包含口頭報告、書面報告與專案檔，同時填寫每組員的貢獻度描述與百分比。
2. 教授開啟評分模組。
3. 在課堂展示時，教授依照組別順序輸入展示分數。包含全組基本分與個人加減分。打分數時，教授看得到學生的貢獻度描述。老師可以對個別組員及展示寫備註。
4. 匯出學生的總成績，包含學號，姓名，組別，報告分數與展示分數，與貢獻度描述。

## 4. 身份驗證與登入 (Authentication)
為了確保教學使用方便且安全，建議採用以下方案：

*   **預設學號登入**
    *   **帳號**：學號。
    *   **初始密碼**：學號最後 4 碼。
    *   **強制要求**：第一次登入後必須修改密碼。

**並在學生清單匯入時自動產生帳密。**

## 5. 技術架構建議
- **前端**：React (Vite) + Tailwind CSS (打造現代化的 UI)。
- **後端/資料庫**：django, mysql。
- **部署**： Railway, Supabase, docker。

## 5. 資料結構預覽
- `users`: `{ uid, studentId, name, role }`
- `groups`: `{ groupId, groupName, members: [studentIds], proposalUrl, midTermStatus, finalScore }`
